# 설문 공고 카드 — 데이터 양식

새 조사(공고)를 추가할 때 아래 필드를 가진 **객체 하나**를 `app.js`의 `SAMPLE_LISTINGS` 배열에 넣거나, 브라우저 개발자 도구 콘솔에서 `localStorage`의 `sp_listings`에 반영하면 됩니다.

---

## 필수 필드

| 필드 | 타입 | 설명 | 예시 |
|------|------|------|------|
| **id** | `number` | 공고 고유 번호 (중복 불가) | `7` |
| **title** | `string` | 설문 제목 (카드 제목에 표시) | `'2026 MZ세대 금융 생활 실태조사'` |
| **description** | `string` | 설문 설명 (카드 본문, 2줄까지 표시) | `'20~30대의 금융 생활 패턴과...'` |
| **reward** | `number` | 사례비 (원) | `5000` |
| **surveyLink** | `string` | 설문 참여 링크 (전체 URL) | `'https://forms.gle/xxxxx'` |
| **deadline** | `string` | 마감일 (`YYYY-MM-DD`) | `'2026-04-15'` |
| **status** | `string` | (선택) 수동 상태값. 실제 노출 상태는 `deadline`/`maxParticipants` 기준으로 자동 계산됨 | `'active'` 또는 `'closed'` |
| **category** | `string` | 카테고리 (배지로 표시) | `'금융'`, `'외식·F&B'`, `'자동차'` 등 |
| **estimatedTime** | `string` | 예상 소요 시간 | `'15분'`, `'20분'` |
| **maxParticipants** | `number` | 모집 인원 (명). 카드 푸터에 "모집 N명"으로 표시 | `30`, `100` |

---

## 카드에서 쓰이는 위치

- **헤더**: `category`(배지), 마감 상태 배지(데드라인 초과 또는 모집인원 마감 시 자동 마감)
- **본문**: `title`, `description`(2줄 말줄임)
- **푸터**: `estimatedTime`, `maxParticipants`(`현재참여자/최종모수` 형식), `reward`(원), 설문 참여/완료 버튼(`surveyLink`, 완료 여부)

---

## JSON 예시 (새 공고 1개)

```json
{
  "id": 7,
  "title": "새 조사 제목",
  "description": "이 설문은 ~를 파악하기 위한 조사입니다. 참여 조건은 ... 입니다.",
  "reward": 3000,
  "surveyLink": "https://forms.gle/실제_폼_링크",
  "deadline": "2026-05-15",
  "status": "active",
  "category": "기타",
  "estimatedTime": "10분",
  "maxParticipants": 30
}
```

## app.js에 직접 추가하는 방법

`40. 신사업팀 (바이브코딩)/설문패널/app.js` 파일에서 `SAMPLE_LISTINGS` 배열을 찾아, 마지막 항목 뒤에 쉼표를 붙이고 위 형식으로 객체를 하나 추가한 뒤 저장하면 됩니다.  
(초기 로드 시 `sp_listings`가 없을 때만 이 샘플이 사용되므로, 이미 사용 중인 환경에서는 localStorage의 `sp_listings`를 수정하거나, 코드에서 `getStore(KEYS.listings)`가 반환하는 배열에 새 항목을 넣는 방식으로 추가해야 합니다.)
